# Security Policy

## Supported Versions

Currently, only the latest release of `rllm` receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public GitHub issue.

Instead, use [GitHub Security Advisories](https://github.com/giovannifil-64/rllm/security/advisories/new) to privately disclose the vulnerability.

We will review your report and respond as soon as possible. We will work with you to evaluate the risk, develop a fix, and coordinate public disclosure.

Please include in your report:
- Type of vulnerability (e.g., path traversal, unsafe deserialization, dependency with known CVE)
- Step-by-step instructions to reproduce it
- Potential impact on users or systems
